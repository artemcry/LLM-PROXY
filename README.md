# LLMProxy

A lightweight proxy server that intercepts OpenAI-compatible chat completion requests (e.g. from [Cline](https://github.com/clinebot/cline)) and transparently substitutes image content with text descriptions generated by Gemini Vision — enabling vision-capable workflows with backends that do not natively support image inputs, such as MiniMax.

## How It Works

```
Client (images) → LLMProxy → Gemini Vision (description) → MiniMax (response)
```

1. Intercepts `POST /v1/chat/completions` and `POST /chat/completions`
2. Detects `image_url` / `image` parts in the message array
3. Fetches image data (base64 data URI or remote URL)
4. Sends each image to Gemini Vision and retrieves a textual description
5. Replaces image parts with `[Image description by Gemini Vision]:\n<description>`
6. Forwards the modified request to MiniMax and streams or returns the response
7. Caches descriptions by SHA-256 hash (in-memory LRU, 256 entries by default)

## Requirements

- Python 3.10+
- MiniMax API key
- Google Gemini API key

## Installation

```bash
# Clone the repository
git clone https://github.com/artemcry/LLM-PROXY.git
cd LLM-PROXY

# Copy and fill in the environment file
cp .env.example .env
# Set MINIMAX_API_KEY and GEMINI_API_KEY

# Install dependencies
pip install -r requirements.txt
# or manually: pip install fastapi uvicorn httpx google-genai python-dotenv

# Start the server
python server_bg.py   # background mode — logs written to logs/
# or
python server.py      # foreground mode — logs to stdout
```

The server listens on the host and port defined by `SERVER_HOST` / `SERVER_PORT` in `.env` (default: `http://127.0.0.1:8087`).

## Client Configuration (Cline)

In Cline settings (`.cline/settings.json` or via the UI):

```json
{
  "openaiApiKey": "dummy",
  "openaiBaseUrl": "http://127.0.0.1:8087/v1"
}
```

The API key value is not validated by the proxy — any non-empty string will work. The original `Authorization: Bearer` header is forwarded as-is to MiniMax.

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `MINIMAX_API_KEY` | **required** | MiniMax API key |
| `GEMINI_API_KEY` | **required** | Google Gemini API key |
| `SERVER_HOST` | `127.0.0.1` | Host address to bind |
| `SERVER_PORT` | `8087` | Port to listen on |
| `MINIMAX_URL` | `https://api.minimax.io/v1/chat/completions` | MiniMax endpoint |
| `MINIMAX_MODEL` | `MiniMax-M2.7` | MiniMax model ID |
| `GEMINI_MODEL` | `gemini-2.5-flash` | Gemini model used for image description |
| `IMAGE_CACHE_MAX` | `256` | Maximum number of cached image descriptions |

## Image Description Cache

Descriptions are cached by the SHA-256 hash of the raw image bytes. The cache is in-memory and uses an LRU eviction policy — the oldest entry is dropped when the limit is reached. The cache is cleared on server restart.

## Error Handling

| Condition | Behavior |
|---|---|
| Gemini `429 / RESOURCE_EXHAUSTED` | Returns a user-facing quota error message; the request is not forwarded to MiniMax |
| Network or decoding errors | Image part is replaced with `[Image could not be described: <error>]`; the request continues normally |
